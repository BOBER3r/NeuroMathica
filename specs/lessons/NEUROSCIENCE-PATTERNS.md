# Neuroscience Patterns for Math Learning — NeuroMathica

> Reference document for lesson designers. Every design decision should
> trace back to one of these evidence-based patterns.

---

## 1. The Neural Learning Sequence (NLS)

The 7-stage sequence is designed around how the brain actually learns
mathematical concepts, not how textbooks traditionally present them.

### Why This Order Works

```
Traditional:  Definition → Example → Practice → Test
NeuroMathica: Hook → Spatial → Discovery → Symbol → Real World → Practice → Reflection

The key difference: SPATIAL BEFORE SYMBOLIC
```

**Evidence**: Dehaene (2011) showed that mathematical understanding is
grounded in spatial representations in the intraparietal sulcus (IPS).
Students who build spatial intuition FIRST and then map symbols onto it
show 40% better transfer to novel problems than those who start with
symbols.

## 2. Core Patterns

### Pattern 1: Curiosity Before Content (Hook)

**Brain region**: Ventral Tegmental Area (VTA) → dopamine release
**Mechanism**: Novel, unexpected stimuli activate the dopamine system,
which primes the hippocampus for memory formation.
**Application**: Every lesson starts with a "wow" moment — an animation
that creates a question the student WANTS to answer.

**Design Rules**:
- Hook must create a QUESTION, not give an answer
- Duration: 10-60 seconds (enough to engage, not enough to bore)
- No interaction required (reduce cognitive load)
- Visual quality must be high (signals "this is worth paying attention to")

---

### Pattern 2: Embodied Cognition (Spatial Experience)

**Brain region**: Intraparietal Sulcus (IPS) + Motor Cortex
**Mechanism**: When students physically manipulate mathematical objects
(drag, rotate, resize), motor cortex activation creates stronger memory
traces than passive observation.
**Application**: Every concept has a physical/spatial representation
that students interact with BEFORE seeing any notation.

**Design Rules**:
- The spatial model must be THE metaphor for the concept:
  - Numbers → Number line (position = value)
  - Fractions → Bars/circles (parts of whole)
  - Angles → Rotation of rays
  - Factors → Rectangle arrays of dots
  - Addition → Walking/movement on number line
- Interaction must change the visual IN REAL TIME (no "submit and wait")
- Minimum 8-10 interactions before allowing progression
- The student should discover constraints naturally (what happens at 10 ones?)

---

### Pattern 3: Guided Discovery (Not Telling)

**Brain region**: Prefrontal Cortex + Anterior Cingulate Cortex
**Mechanism**: Insights discovered by the student trigger stronger
dopamine responses than information given by the teacher. The ACC
monitors "prediction errors" — when what you expect doesn't match
what happens, learning is enhanced.
**Application**: Prompts guide attention but don't reveal the answer.

**Design Rules**:
- Each prompt should direct ATTENTION, not give INFORMATION
- "Notice that..." not "The rule is..."
- "What happens when..." not "When you do X, Y happens"
- The student should feel like THEY discovered it
- 3-6 prompts, each building on the last
- Allow productive struggle (15-30 seconds per prompt before hint)

---

### Pattern 4: Symbol Grounding (Symbol Bridge)

**Brain region**: Angular Gyrus (symbol ↔ meaning mapping)
**Mechanism**: Mathematical notation is meaningful only when it's
connected to a mental model. The angular gyrus maps symbols to their
spatial/conceptual representations. Without this grounding, notation
is just "marks on paper."
**Application**: Notation appears ON TOP of the spatial model, with
explicit visual connections (arrows, highlights, color-coding).

**Design Rules**:
- Notation appears ONE piece at a time (not a wall of formulas)
- Each piece has a visual connection to the spatial model
- Colors match between notation and the spatial elements
- Duration: 1.5-2 seconds between each notation appearance
- Student should be able to "see" what each symbol means

---

### Pattern 5: Episodic Anchoring (Real World)

**Brain region**: Hippocampus (episodic memory)
**Mechanism**: Abstract concepts connected to personal experience are
remembered 3-5x longer than isolated facts. The hippocampus stores
context-rich memories that serve as retrieval cues.
**Application**: 3-4 real-world scenarios that connect to student's life.

**Design Rules**:
- Scenarios must be relevant to 11-14 year olds
- Money, games, food, sports, social media > "adult" examples
- Each scenario should make the student think "oh, THAT'S what this is"
- Brief (1-2 sentences per scenario)

---

### Pattern 6: Retrieval Practice (Practice)

**Brain region**: Basal Ganglia (procedural memory) + Hippocampus
**Mechanism**: Actively retrieving information from memory strengthens
the neural pathways far more than re-reading or re-watching. Testing
IS learning, not just assessment.
**Application**: 9+ problems across 3 cognitive layers.

**The Three Layers**:
```
Layer 0 — RECALL:     Can you STATE the fact/formula?
Layer 1 — PROCEDURE:  Can you EXECUTE the method?
Layer 2 — UNDERSTANDING: Can you EXPLAIN why it works?
```

**Design Rules**:
- 3 problems per layer minimum
- Interleave layers (don't do all recall then all procedure)
- IRT difficulty: Layer 0 = easy (-1.5 to -0.5), Layer 1 = medium (0 to 1), Layer 2 = hard (1 to 2)
- NO free-text for graded answers (can't validate)
- Feedback stays visible until student taps "Next"
- Every correct answer should feel rewarding (color, animation)
- Every incorrect answer should be instructive (brief explanation)

---

### Pattern 7: Self-Explanation Effect (Reflection)

**Brain region**: Prefrontal Cortex (metacognition) + Default Mode Network
**Mechanism**: Explaining a concept in your own words forces the brain
to organize and consolidate the learned information. Students who
self-explain show 2-3x better retention and transfer.
**Application**: Open-ended prompt asking student to explain the concept.

**Design Rules**:
- Prompt should be SPECIFIC, not "what did you learn?"
- "Explain WHY [specific thing] works" is better than "explain [concept]"
- Not graded (participation credit only)
- Minimum character count to encourage effort (20-30 chars)
- "Skip" option available but de-emphasized
- Encouraging feedback regardless of quality

## 3. Emotional Design Patterns

### The Emotional Arc

```
Hook:       Curiosity / Wonder     (dopamine spike)
Spatial:    Engagement / Flow      (sustained dopamine)
Discovery:  Productive Struggle    (ACC activation)
            → Insight / Aha!       (dopamine burst)
Symbol:     Confidence             (competence signal)
Real World: Relevance / Connection (hippocampal binding)
Practice:   Mastery / Growth       (procedural fluency)
Reflection: Pride / Ownership      (metacognitive reward)
```

### Anti-Patterns to Avoid
- ❌ Timer pressure (cortisol blocks hippocampal encoding)
- ❌ Punishment for wrong answers (amygdala activation → learning shutdown)
- ❌ Too many failures in a row (>3 = frustration → disengage)
- ❌ Too many successes in a row (>5 = boredom → disengage)
- ❌ Wall of text (working memory overload)
- ❌ Speed rewards (encourages memorization over understanding)

### Patterns to Use
- ✅ Immediate visual feedback (dopamine for every interaction)
- ✅ Progressive difficulty within practice (flow state)
- ✅ Celebration on insight moments (not just correct answers)
- ✅ Connection to prior concepts (strengthens neural networks)
- ✅ Physical/spatial metaphors (motor cortex = stronger encoding)
- ✅ Student agency (choice → engagement)

## 4. Visual Interaction Patterns

### Interaction Types (ranked by learning effectiveness)

| Type | Encoding Strength | Use When |
|------|-------------------|----------|
| Drag to manipulate | ★★★★★ | Spatial exploration (number lines, shapes) |
| Drag to arrange | ★★★★☆ | Ordering, expanded form, sorting |
| Tap to toggle | ★★★☆☆ | Shading fraction parts, selecting options |
| Tap to select | ★★★☆☆ | Multiple choice, identification |
| Numeric input | ★★☆☆☆ | Single-number answers |
| Slider | ★★☆☆☆ | Adjusting parameters (denominator, zoom) |

### Feedback Patterns

| Event | Visual | Timing |
|-------|--------|--------|
| Tap/click | Scale pulse (0.95 → 1.0) | Instant |
| Correct answer | Green flash + checkmark | 200ms |
| Incorrect answer | Red shake + brief explanation | 300ms + stays |
| Drag snap | Spring to position | 150ms spring |
| Stage complete | Celebration particles | 500ms |
| Insight moment | Glow + scale up + hold | 800ms |

### Color Semantics
```
Correct / Positive:  #34d399 (emerald)
Incorrect / Error:   #fb7185 (rose)
Active / Selected:   #818cf8 (indigo)
Highlight / Insight: #fbbf24 (amber)
Info / Neutral:      #60a5fa (blue)
Disabled:            #475569 (gray)
```
